3.1 白化(Whitening)

  由于原始图像相邻像素值具有高度相关性，所以图像数据信息冗余。数据白化主要有两个作用：（1）减少特征之间的相关性，（2）特征具有相同的方差（协方差阵为1）。一般情况下，所获得的数据都具有相关性，所以通常都要求对数据进行初步的白化或球化处理，因为白化处理可去除各观测信号之间的相关性，从而简化了后续独立分量的提取过程，而且，通常情况下，数据进行白化处理与不对数据进行白化处理相比，算法的收敛性较好。

Python——可调用对象callable object

发表于 2019-11-20 更新于 2019-11-29 分类于 编程语言

1. 什么是可调用对象(callable object)

  • 定义：但凡是可以把一对括号()应用到某个对象上，都可以称之为可调用对象
  • 内涵：我们平时自定义的函数、内置函数和类都属于可调用对象。
      外延：如果在类中实现了__call__方法，那么该类的实例对象也将成为一个可调用对象，其行为可以表现得像一个函数一样。允许一个类的实例对象像函数一样被调用，实际上意味着obj()和obj.call()是相通的。

后端开发——概念区分

发表于 2019-11-20 分类于 后端开发

1. 基本概念

1.1 服务器(Server)：
  • 服务器serve指的是一段程序，或者为客户提供该程序的设备。包括：
  • application server
  • file server
  • computing server
  • database server
  • mail server
  • proxy server
  • web server
1.2 Web服务(Web service)：

  Web服务是一个概念，广义上是指一台设备为其他设备提供的一种服务，狭义上指运行在一台电脑上的一个server。

1.3 Web服务器(Web server)：
  • Web服务器是在运行在物理服务器上的一个程序（也可以指该物理服务器），它永久地等待客户端（主要是浏览器，比如Chrome，Firefox等）发送请求request。
  • 当收到请求之后，它会生成相应的响应并将其返回至客户端。
  • Web服务器通过HTTP协议与客户端通信，因此也被称为HTTP服务器。
  • web服务器可以被嵌入到一个打印机、路由器、网络摄像头等设备中。
  • 因特网上常见的web服务器有apache, nginx, IIS等。
1.4 Web框架(Web framework)
  • Web框架也称作Web应用框架(Web application framework)，指的是支持各种web application（包括web services, web resources, and web APIs等）的软件框架。
  • 一种比较经典的框架采用MVC架构
  • Django
  • flask
  • spring MVC
  • ASP.NET core
1.5 Web应用(Web application)

  Web 服务器接受 Http Request，返回 Response，很多时候 Response 并不是静态文件，因此需要有一个应用程序根据 Request 生成相应的 Response。这里的应用程序主要用来处理相关业务逻辑，读取或者更新数据库，根据不同 Request 返回相应的 Response。注意这里并不是 Web 服务器本身来做这件事，它只负责 Http 协议层面和一些诸如并发处理，安全，日志等相关的事情。应用程序可以用各种语言编写（Java, PHP, Python, Ruby等），这个应用程序会从Web服务器接收客户端的请求，处理完成后，再返回响应给Web服务器，最后由Web服务器返回给客户端。运行过程为：

2. 逻辑关系

2.1 Web服务器和Web框架的关系

  Web服务器和Web框架之间需要进行通信，于是需要设计一套双方都遵守的接口。对于Python来说，这个接口是Web服务器网关接口WSGI(Web Server Gateway Interface)。其他编程语言也拥有类似的接口：例如Java的Servlet API和Ruby的Rack。
  Python WSGI的出现，让开发者可以将 Web 框架与 Web 服务器的选择分隔开来，不再相互限制。现在，你可以真正地将不同的 Web 服务器与Web框架进行混合搭配，选择满足自己需求的组合。例如，可以使用 Gunicorn 或Nginx/uWSGI来运行Django、Flask或web.py应用。
