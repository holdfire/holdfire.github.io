3.1 白化(Whitening)

  由于原始图像相邻像素值具有高度相关性，所以图像数据信息冗余。数据白化主要有两个作用：（1）减少特征之间的相关性，（2）特征具有相同的方差（协方差阵为1）。一般情况下，所获得的数据都具有相关性，所以通常都要求对数据进行初步的白化或球化处理，因为白化处理可去除各观测信号之间的相关性，从而简化了后续独立分量的提取过程，而且，通常情况下，数据进行白化处理与不对数据进行白化处理相比，算法的收敛性较好。

1. 什么是可调用对象(callable object)

  • 定义：但凡是可以把一对括号()应用到某个对象上，都可以称之为可调用对象
  • 内涵：我们平时自定义的函数、内置函数和类都属于可调用对象。
      外延：如果在类中实现了__call__方法，那么该类的实例对象也将成为一个可调用对象，其行为可以表现得像一个函数一样。允许一个类的实例对象像函数一样被调用，实际上意味着obj()和obj.call()是相通的。

后端开发——概念区分

发表于 2019-11-20 分类于 后端开发

1. 基本概念

1.1 服务器(Server)：
  • 服务器serve指的是一段程序，或者为客户提供该程序的设备。包括：
  • application server
  • file server
  • computing server
  • database server
  • mail server
  • proxy server
  • web server
1.2 Web服务(Web service)：

  Web服务是一个概念，广义上是指一台设备为其他设备提供的一种服务，狭义上指运行在一台电脑上的一个server。

1.3 Web服务器(Web server)：
  • Web服务器是在运行在物理服务器上的一个程序（也可以指该物理服务器），它永久地等待客户端（主要是浏览器，比如Chrome，Firefox等）发送请求request。
  • 当收到请求之后，它会生成相应的响应并将其返回至客户端。
  • Web服务器通过HTTP协议与客户端通信，因此也被称为HTTP服务器。
  • web服务器可以被嵌入到一个打印机、路由器、网络摄像头等设备中。
  • 因特网上常见的web服务器有apache, nginx, IIS等。
1.4 Web框架(Web framework)
  • Web框架也称作Web应用框架(Web application framework)，指的是支持各种web application（包括web services, web resources, and web APIs等）的软件框架。
  • 一种比较经典的框架采用MVC架构
  • Django
  • flask
  • spring MVC
  • ASP.NET core
1.5 Web应用(Web application)

  Web 服务器接受 Http Request，返回 Response，很多时候 Response 并不是静态文件，因此需要有一个应用程序根据 Request 生成相应的 Response。这里的应用程序主要用来处理相关业务逻辑，读取或者更新数据库，根据不同 Request 返回相应的 Response。注意这里并不是 Web 服务器本身来做这件事，它只负责 Http 协议层面和一些诸如并发处理，安全，日志等相关的事情。应用程序可以用各种语言编写（Java, PHP, Python, Ruby等），这个应用程序会从Web服务器接收客户端的请求，处理完成后，再返回响应给Web服务器，最后由Web服务器返回给客户端。运行过程为：

2. 逻辑关系

2.1 Web服务器和Web框架的关系

  Web服务器和Web框架之间需要进行通信，于是需要设计一套双方都遵守的接口。对于Python来说，这个接口是Web服务器网关接口WSGI(Web Server Gateway Interface)。其他编程语言也拥有类似的接口：例如Java的Servlet API和Ruby的Rack。
  Python WSGI的出现，让开发者可以将 Web 框架与 Web 服务器的选择分隔开来，不再相互限制。现在，你可以真正地将不同的 Web 服务器与Web框架进行混合搭配，选择满足自己需求的组合。例如，可以使用 Gunicorn 或Nginx/uWSGI来运行Django、Flask或web.py应用。

Notation:

  • 目标检测框的参数：中心点坐标为(bx,by)(b_x, b_y)(bx​,by​)，框的高度和宽度分别为bh,bwb_h, b_wbh​,bw​
  • 目标检测标签：[pc,bx,by,bh,bw,c1,c2,...,cn][p_c,b_x, b_y,b_h, b_w, c_1,c_2,...,c_n][pc​,bx​,by​,bh​,bw​,c1​,c2​,...,cn​]，其中，pcp_cpc​表示样本是否包含目标，c1,c2,...,cnc_1,c_2,...,c_nc1​,c2​,...,cn​表示样本包含类别目标的个数。

  用convnet做人体姿态估计，数据的标注方法同上。

3.3 目标检测object detection

  基于滑动窗口的目标检测方法(sliding windows detection)

3.4 convolutional implementation of sliding windows

  用卷积层代替全连接层，不用预先切割窗口出来，最后得到的卷积层上可以得到边界框bounding box的位置，但可能不够准确。

3.5 边界框预测Bounding box prediction

YOLO算法：

  • 先把整张图像分成n×nn \times nn×n个网格；
  • 然后对每个网格都生成一个目标检测的标签，如果一个对象在多个网格中，该对象属于其中心所在的那个网格；
  • 需要注意：此时该对象的标签中的目标框参数需要改成以该网格的左上角为原点了。bx,byb_x,b_ybx​,by​仍然在0和1之间，但bh,bwb_h,b_wbh​,bw​可以大于1，这表明待检测物体有部分在该网格外面。

3.6 交并比 Intersection over Union

  交并比IoU被用来衡量目标检测框结果的好坏，通常约定IoU大于0.5，就认为目标检测正确，其计算公式为：

IoU=A∩BA∪BIoU = \frac {A \cap B} {A \cup B} IoU=A∪BA∩B​

3.7 非极大值抑制 Non-Maxiumum Suppression

  模型可能会对同一个对象做出多次检测，非极大值抑制可以清理这些检测结果，这样每个目标只做一次检测。算法原理：

  • （1）对每个网格做出预测，抛弃那些pc≤0.6p_c \leq 0.6pc​≤0.6的低概率边界框；
  • （2）比较每个检测结果的概率pcp_cpc​，找到最大的那个并认为是最有可能的；
  • （3）然后逐一审视剩下的矩形，那些和这个最大边界框有很高的交并比的框会被抑制；循环步骤3和4直至无边界框剩余
  • （4）最后抛弃那些被抑制的框，得到的就是最终的目标检测结果。
    如果要检测多个类别，应该对每个框的每个类别分别进行非极大值抑制。

3.8 Anchor Boxes

  假设我们使用了2种尺寸的anchor box，那么在每个网格中样本标签为y=[pc,bx,by,bh,bw,c1,c2,c3,pc,bx,by,bh,bw,c1,c2,c3]y = [p_c, b_x, b_y,b_h, b_w, c_1,c_2,c_3, p_c, b_x, b_y,b_h, b_w, c_1,c_2,c_3]y=[pc​,bx​,by​,bh​,bw​,c1​,c2​,c3​,pc​,bx​,by​,bh​,bw​,c1​,c2​,c3​]
通常可以选择5-10种不同形状和大小的anchor box。
算法不好处理的情况1：如果一个网格中有3个对象，但是只有2个anchor box
算法不好处理的情况2：同一个网格中含有2个对象，且它们的anchor box尺寸一样

3.9 YOLO算法

  假设我们的目标检测任务为：

  • 要检测3种对象：行人，汽车和摩托；
  • 将图片划分为3×33 \times 33×3的网格；
  • 使用2个的anchor boxes，那么我们在每个网格的标签就是：y=[pc,bx,by,bh,bw,c1,c2,c3,pc,bx,by,bh,bw,c1,c2,c3]y = [p_c, b_x, b_y,b_h, b_w, c_1,c_2,c_3, p_c, b_x, b_y,b_h, b_w, c_1,c_2,c_3]y=[pc​,bx​,by​,bh​,bw​,c1​,c2​,c3​,pc​,bx​,by​,bh​,bw​,c1​,c2​,c3​]
  • 那么每张图片的标签就是3×3×2×83 \times 3 \times 2 \times 83×3×2×8维的；

3.10 候选区域 Region Proposal

  R-CNN网络不对每个滑动窗口做卷积，而是选出一些区域做卷积，选取的方法是：先对图像做分割，根据分割边缘找出约2000个候选区域，然后在这些候选区域上放置边界框跑分类。
  Fast R-CNN网络：用卷积实现滑窗来做分类任务，但得到候选区域的聚类步骤仍然很慢
  Faster R-CNN网络：使用卷积神经网络，而不是传统的分割算法来获得候选区域。

2. 边缘提取——HED网络

  整体嵌套边缘检测(Holistically-Nested Edge Detection)是一个基于深度卷积神经网络的图像边缘提取算法。其特点是：（1）整体图像训练和预测；（2）多尺度和多层特征学习。该深度模型利用全卷积神经网络，自动学习丰富的分层表示。（基于侧面响应的深层监督指导）

2.1 多尺度深度学习

  多尺度深度学习可以分为四类，即多流学习(multi-stream learning)、跳网(skip-net learning)、多输入单模型以及独立网训练，如下图所示：（a）多流架构；（b）跳网架构；（c）多尺度输入的单一模型；（d）不同网络独立训练；（e）HED，其中添加多个侧面输出。

  在表示和计算方面，前四种方法存在冗余，HED是一个相对简单的变体，能够从多个尺度产生预测。该结构可以被解释为（d）独立网络的整体嵌套版，包括多个侧输出单流深度网络。隐藏层监督可以改善图像分类任务的优化和泛化。如果需要统一输出，多侧向输出可以灵活地添加额外的融合层。

3. 边缘提取——CASENet网络

  CASENet网络将每个边缘像素与多于一个的边缘类相关联，是一种基于ResNet的端到端深度语义边缘学习的架构，同时也是一种跳层(skip-layer)架构，其中顶层卷积层的类别边缘激活共享并与底层特征的同一集融合，用一种多标签损失函数来监督激活的融合。

4. 轮廓提取——DeepEdge

  DeepEdge利用目标相关特征作为轮廓的高级线索检测。设计了一个多级深度网络，由五个卷积层和一个分叉全连接子网络(bifurcated fully-connected subnetwork)组成。

5. 轮廓提取——CEDN
